# Google广告接入方案

## 1. 广告类型选择

针对HTML5游戏平台，建议接入以下几种Google广告类型：

1. **展示广告（Display Ads）**：
   - 横幅广告（Banner ads）：放置在页面顶部、底部或侧边栏
   - 信息流广告（Native ads）：融入游戏列表中，与内容相似的广告形式
   - 响应式广告单元：自动适应不同设备屏幕大小

2. **插页式广告（Interstitial Ads）**：
   - 游戏加载前显示的全屏广告
   - 在用户浏览页面之间的切换时展示
   - 用户开始新会话时展示

## 2. 广告放置位置

以下是详细的广告位置布局规划：

### 2.1 全站通用广告位

1. **顶部横幅广告**：
   - 位置：页面顶部导航栏下方，主内容区域之前
   - 尺寸：728x90（桌面端）/ 320x50（移动端）
   - 所有页面共用，统一ID

2. **底部横幅广告**：
   - 位置：主内容区域之后，页脚之前
   - 尺寸：728x90（桌面端）/ 320x50（移动端）
   - 所有页面共用，统一ID

3. **侧边栏广告**：
   - 位置：右侧边栏顶部和中部
   - 尺寸：300x250 或 300x600
   - 适用于有侧边栏的页面

### 2.2 页面特定广告位

#### 首页广告位

1. **热门游戏区域广告**：
   - 位置：热门游戏列表中第3和第8个位置
   - 尺寸：与游戏卡片相同（响应式）
   - 标记：带有"广告"标识，与内容区分

2. **推荐游戏区域广告**：
   - 位置：推荐游戏列表之后
   - 尺寸：970x250（桌面）/ 300x250（移动）
   - 样式：响应式横幅

3. **分类导航区域广告**：
   - 位置：游戏分类导航下方
   - 尺寸：728x90
   - 仅桌面端显示

#### 游戏详情页广告位

1. **游戏信息下方广告**：
   - 位置：游戏描述和特性列表之后，评论区之前
   - 尺寸：728x90（桌面）/ 300x250（移动）
   - 高度相关性广告单元

2. **相关游戏区域广告**：
   - 位置：相关游戏推荐列表中，每5个游戏后插入1个广告
   - 尺寸：与游戏卡片相同
   - 标记为"赞助内容"

3. **右侧边栏推荐广告**：
   - 位置：游戏信息右侧边栏
   - 尺寸：300x600
   - 仅桌面端显示
   - 可粘性滚动（sticky）

#### 游戏分类页广告位

1. **分类标题下方广告**：
   - 位置：分类描述之后，游戏列表之前
   - 尺寸：728x90
   - 所有分类页面通用

2. **游戏列表内广告**：
   - 位置：分类游戏列表中，每8个游戏后插入广告
   - 尺寸：与游戏卡片相同
   - 响应式布局

#### 搜索结果页广告位

1. **搜索结果顶部广告**：
   - 位置：搜索条件/筛选器下方，结果列表之前
   - 尺寸：728x90（桌面）/ 300x250（移动）
   - 样式：与搜索结果区分明显

2. **搜索结果列表内广告**：
   - 位置：搜索结果列表中，第3、第10、第20位
   - 尺寸：与搜索结果条目相同
   - 标记为"广告"

#### 用户中心页面广告位

1. **非会员用户个人资料页广告**：
   - 位置：用户资料信息下方
   - 尺寸：728x90
   - 包含会员升级提示

2. **游戏历史记录页面广告**：
   - 位置：游戏历史列表底部
   - 尺寸：300x250
   - 仅对非会员用户显示

### 2.3 插页式广告触发点

1. **游戏加载前广告（iframe实施方案）**：
   - 触发时机：用户点击"开始游戏"按钮后，iframe加载游戏URL前
   - 尺寸：全屏插页式广告
   - 配置：倒计时5秒后显示跳过按钮，展示广告内容
   - 实现方式：在iframe加载游戏URL前展示广告，广告结束后再加载游戏

2. **页面切换广告**：
   - 触发时机：从游戏列表页导航到游戏详情页
   - 频率控制：同一用户15分钟内最多显示1次
   - 尺寸：全屏
   - 实现方式：在页面加载时检查用户最近是否已看过此类广告

3. **会话开始广告**：
   - 触发时机：非会员用户每次会话首次浏览游戏时
   - 频率控制：每24小时最多显示2次
   - 包含会员升级提示
   - 实现方式：使用会话存储和本地存储跟踪展示频率

## 3. 技术实现方案

1. **Google AdSense/AdManager集成**：
   - 在base.html的head部分添加Google AdSense脚本
   - 在需要显示广告的位置创建广告容器

2. **广告加载控制**：
   - 创建专门的广告管理模块，控制广告展示逻辑
   - 实现广告延迟加载机制，提高页面加载速度

3. **会员与广告的关系**：
   - 为会员提供无广告体验
   - 在settings.py中添加广告控制配置，根据用户会员状态决定是否显示广告

4. **会员无广告体验**：
   - 会员用户不显示任何广告（二元逻辑：有会员=无广告，无会员=有广告）
   - 会员状态检测逻辑：实时检查用户订阅状态，确保订阅过期后恢复广告展示
   - 会员标识：为会员用户添加会话标识，避免每次请求都查询数据库

## 4. 代码实现步骤

1. **创建广告配置模块**：
   - 在settings.py中添加Google AdSense配置项
   - 定义简洁的广告展示策略，基于会员状态（是/否）控制广告展示

2. **创建广告模板标签**：
   - 开发自定义模板标签，方便在不同位置灵活插入广告
   - 支持根据用户会员状态控制广告展示
   - 添加会员状态检测逻辑，会员用户不加载广告

3. **开发广告统计分析功能**：
   - 跟踪广告展示和点击数据
   - 分析不同页面、不同位置的广告效果
   - 监控会员转化率与广告收益关系

4. **游戏加载前广告实现**：
   - 在游戏详情页添加广告容器和控制逻辑
   - 使用JavaScript控制广告展示和iframe游戏加载流程
   - 为会员用户跳过广告直接加载游戏
   - 实现iframe加载控制，确保广告结束后才加载游戏内容

5. **页面切换广告实现**：
   - 使用会话存储记录用户的广告展示历史
   - 在页面加载时检查最近广告展示时间，控制频率
   - 添加适当的过渡动画，提高用户体验

6. **会话开始广告实现**：
   - 使用本地存储跟踪用户24小时内的广告展示次数
   - 在会话开始时检查用户会员状态和广告展示次数
   - 结合会员升级推广，增加会员转化机会

## 5. 用户体验优化

1. **广告加载性能优化**：
   - 异步加载广告，不阻塞页面渲染
   - 针对移动设备优化广告展示

2. **广告内容相关性**：
   - 配置Google AdSense，展示与游戏内容相关的广告
   - 根据用户兴趣和历史行为展示个性化广告

3. **广告与内容平衡**：
   - 控制每个页面的广告数量，避免过度干扰用户
   - 优化广告位置，确保不遮挡重要内容

4. **会员价值宣传**：
   - 在广告位置为非会员用户展示"升级为会员可移除广告"的提示
   - 在广告密集区域突出展示会员特权

## 6. 合规性和隐私

1. **隐私政策更新**：
   - 更新网站隐私政策，包含Google广告追踪相关信息
   - 实现cookie同意机制，符合GDPR等隐私法规

2. **广告内容控制**：
   - 配置AdSense过滤器，避免不适合的广告内容
   - 针对不同地区用户展示符合当地法规的广告

## 7. 实施路线图

1. **前期准备阶段**：
   - 注册Google AdSense账号并获取发布商ID
   - 在Google广告平台创建广告单元
   - 更新网站隐私政策和用户协议
   - 设计会员无广告体验的技术实现方案

2. **开发阶段**：
   - 在settings.py中添加广告配置
   - 开发广告模板标签和管理模块，包含会员状态检测
   - 在关键页面添加广告位和会员升级提示
   - 实现会员无广告体验功能
   - 开发游戏加载前广告功能（iframe实施方案）
   - 实现页面切换广告和会话开始广告功能
   - 开发广告频率控制和用户体验优化措施

3. **测试阶段**：
   - 测试不同设备和浏览器下的广告展示效果
   - 验证会员用户是否正确隐藏所有广告
   - 优化广告加载性能和位置
   - A/B测试不同广告策略的效果和会员转化率
   - 测试游戏加载前广告、页面切换广告和会话开始广告的用户体验
   - 验证广告频率控制机制的有效性

4. **上线及监控阶段**：
   - 正式接入Google AdSense
   - 监控广告收益、会员转化率和用户反馈
   - 分析会员订阅增长与广告收入的平衡关系
   - 持续优化广告展示频率和位置
   - 根据数据持续优化广告策略和会员权益
